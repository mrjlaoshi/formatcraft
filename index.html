关于人民币大写金额

人民币大写金额是中国财务、会计、法律领域的正式书写规范。根据中国人民银行发布的《人民币大写数字规范》，填写票据、合同、收据时必须使用大写汉字书写金额，以防止数字被篡改，保障资金安全。

大写规范要求：数字"1-9"分别用"壹贰叁肆伍陆柒捌玖"表示，"0"写作"零"，十、百、千、万分别写作"拾、佰、仟、万"。金额末尾无角分时须写"整"字，有角无分时角后写"整"，有分时则依次写出角分。

适用场景：银行转账单、财务报销单、合同金额填写、增值税发票、支票填写、工程结算单等一切需要正式书写金额的场合。本工具支持最大金额至万亿级，精确到分。

关于中文数字转换

中文数字是汉字文化圈传统的数字表达方式，使用"零一二三四五六七八九"以及"十、百、千、万、亿"等单位词组合表示数值。中文数字在文学作品、传统文化、正式文件中广泛使用，与阿拉伯数字各有其适用场合。

本工具支持阿拉伯数字与中文小写数字的双向互转，覆盖从个位数到万亿级的整数转换。例如：12345 可转换为"一万二千三百四十五"，反向亦可将"三百六十五"还原为 365。

适用场景：文章写作中数字的中文化表达、古典文学阅读辅助、教育教学、日期与年份的中文书写、传统节日数字表达等场合。如需财务大写请使用"大写金额"工具。

关于 URL 编解码

URL（统一资源定位符）只允许包含特定的 ASCII 字符集。当 URL 中含有中文、空格、特殊符号等字符时，必须经过编码处理才能在网络中正确传输。URL 编码（又称百分号编码）将这些字符转换为 %XX 的十六进制格式，例如"中文"会被编码为 %E4%B8%AD%E6%96%87。

本工具使用标准的 encodeURIComponent 与 decodeURIComponent 函数进行转换，支持完整的 UTF-8 字符集，包括中文、日文、韩文、阿拉伯文等多语言字符的编解码。

适用场景：网页开发中构建包含中文的 URL 参数、API 接口调试、解码浏览器地址栏中的编码字符、处理含特殊字符的查询字符串、爬虫数据解析等开发和运维场景。

关于 Base64 编解码

Base64 是一种将二进制数据转换为可打印 ASCII 字符的编码方案，使用 A-Z、a-z、0-9 及 +、/ 共 64 个字符来表示任意数据。由于其输出仅包含安全的文本字符，Base64 被广泛用于在文本协议（如 HTTP、电子邮件）中传输二进制内容。

本工具支持中文文本的 Base64 编解码（内部采用 UTF-8 编码处理），可双向转换。编码后的 Base64 字符串长度约为原文本的 1.33 倍。需注意 Base64 是编码而非加密，不具备数据保护功能。

适用场景：网页开发中嵌入图片（Data URI）、HTTP Basic 认证头编码、JWT 令牌解析、API 数据传输、电子邮件附件编码、前端存储非文本数据等开发场景。

关于 Unicode 转换

Unicode 是一个覆盖全球所有书写系统的字符编码标准，为每个字符分配了唯一的码点（Code Point）。在编程中，Unicode 字符通常以 \uXXXX 的转义序列表示，其中 XXXX 是四位十六进制数字。例如，汉字"你"的 Unicode 码点为 U+4F60，在代码中写作 \u4f60。

本工具支持将任意文本（包括中文、英文、特殊符号、emoji）转换为对应的 \uXXXX Unicode 转义码，也支持将 Unicode 转义码还原为可读字符。非 ASCII 字符（如中文）会被转换，ASCII 字符（如英文字母）则保持原样不转换。

适用场景：JavaScript、Java、Python 等编程语言中的字符串处理、调试含中文的代码字符串、查询字符对应的 Unicode 码点、解析 JSON 中的转义字符、网络安全中的字符编码分析等。
